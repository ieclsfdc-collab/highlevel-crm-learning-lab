# ChatGPT Instruction Rules for HighLevel CRM Learning

## Purpose

This repo is for MJ's fast HighLevel CRM learning.

The goal is high-velocity, smooth-sailing learning so MJ can confidently navigate HighLevel, create basic records, use correct GHL terminology, and judge whether the AI Orchestrator / Business Systems Investigator needs HighLevel-specific adjustments before a demo to James.

## Teaching Style Required

When ChatGPT provides HighLevel CRM learning steps, the instructions must be clear enough that MJ should not need follow-up questions just to understand what to do next.

Every instruction must be:

- Step-by-step
- Specific
- Action-oriented
- Written using HighLevel terminology first
- Short enough to follow while inside HighLevel
- Clear about expected result
- Clear about how to confirm success
- Clear about what to do if something looks different

## Required Format for Every Exercise

Each exercise must include:

1. Goal
2. Estimated time
3. Starting point
4. What will be created
5. Milestones
6. Exact steps per milestone
7. Expected result per milestone
8. Milestone summary
9. GHL terminology practice
10. Quick self-check
11. Common mistakes to avoid
12. Learning output to save

## Milestone Summary and Reflection Rule

Every exercise must be divided into short milestones.

A milestone should usually take 5 to 15 minutes.

After each milestone, ChatGPT must provide a short learning summary before moving to the next milestone.

The summary must help MJ reflect whether he actually remembers the knowledge transferred, not just whether he clicked the right buttons.

### Required Format After Every Milestone

#### Milestone Summary

In this milestone, you learned that:

- [Short learning point 1]
- [Short learning point 2]
- [Short learning point 3]

#### Remember This in GHL Terms

Use 2 to 4 short sentences using HighLevel terminology.

Example:

A Contact is the central person record in HighLevel. A Tag is used to group, trigger, or identify Contacts. A Custom Field stores extra information on the Contact.

#### Quick Self-Check

Answer these before continuing:

1. What did you just create or inspect?
2. Where can you find it again in HighLevel?
3. What is the correct HighLevel term for it?
4. How is it connected to the lead journey?
5. What could go wrong here that the Orchestrator may need to inspect?

### Continue Rule

Do not continue to the next milestone until the learner can answer the Quick Self-Check.

If the learner is unsure, ChatGPT must explain again using simpler words and one HighLevel example.

## Salesforce Translation Rule

Use HighLevel terminology first. Salesforce comparisons are allowed only after the GHL term is clear.

Correct:

> Create a Contact. In Salesforce terms, this is somewhat similar to a Lead or Contact.

Wrong:

> Create a Salesforce-style Lead.

## High-Velocity Rule

Avoid broad theory unless it directly helps the exercise. Each session should produce one clear practical output inside HighLevel and one short learning note in this repo.
