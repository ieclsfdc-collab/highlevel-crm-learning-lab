# Exercise 03: Create Pipeline and Opportunity

## Goal

Create a simple Pipeline and Opportunity so MJ understands GHL lead/deal tracking.

## Estimated Time

60 to 75 minutes

## Starting Point

Start inside a test sub-account/location.

## What You Will Create

| Item Type | Exact Name to Use |
|---|---|
| Pipeline | MJ Test Pipeline |
| Pipeline Stages | New Lead, Contacted, Booked, Won, Lost |
| Opportunity | Test Opportunity - MJ |

## Milestone 1: Create a Pipeline

### Steps

1. Go to Opportunities.
2. Find Pipelines or Pipeline settings.
3. Create a new Pipeline named `MJ Test Pipeline`.
4. Add these Pipeline stages in order:
   - New Lead
   - Contacted
   - Booked
   - Won
   - Lost
5. Save the Pipeline.

### Expected Result

You can see `MJ Test Pipeline` with five Pipeline stages.

### Milestone Summary

In this milestone, you learned that:

- A Pipeline is the visual process for tracking Opportunities.
- Pipeline stages represent the steps in the lead/deal journey.
- A Pipeline should match how the business follows up and closes leads.

### Remember This in GHL Terms

A Pipeline contains Pipeline stages. An Opportunity moves through the Pipeline as the lead progresses.

### Quick Self-Check

2. What stages did I add?
3. Where can I find the Pipeline again?
4. How does a Pipeline connect to the lead journey?
5. What could the Orchestrator inspect in a Pipeline?

## Milestone 2: Create an Opportunity

### Steps

1. Go to Opportunities.
2. Select `MJ Test Pipeline`.
3. Create a new Opportunity named `Test Opportunity - MJ`.
4. Link it to the Contact `Test Lead - MJ` if available.
5. Place it in the `New Lead` stage.
6. Save the Opportunity.

### Expected Result

You can see `Test Opportunity - MJ` in the `New Lead` stage of `MJ Test Pipeline`.

### Milestone Summary

In this milestone, you learned that:

- An Opportunity is a lead/deal card inside a Pipeline.
- An Opportunity usually connects to a Contact.
- The Opportunity stage shows where the lead/deal is in the process.

### Remember This in GHL Terms

An Opportunity belongs in a Pipeline. A Contact can have an Opportunity that tracks sales or follow-up progress.

### Quick Self-Check

1. What Opportunity did I create?
2. Which Contact is it linked to?
3. Which Pipeline and stage is it in?
4. What is the difference between Contact and Opportunity?
5. What Opportunity issues could the Orchestrator inspect?

## Milestone 3: Move the Opportunity

### Steps

1. Open the Pipeline board.
2. Move `Test Opportunity - MJ` from `New Lead` to `Contacted`.
3. Move it back to `New Lead`.
4. Observe what changes on the board.

### Expected Result

You can manually move an Opportunity between Pipeline stages.

### Milestone Summary

In this milestone, you learned that:

- Opportunities can move between Pipeline stages.
- Stage movement represents business progress.
- Workflows may also create or move Opportunities automatically.

### Remember This in GHL Terms

A Workflow can create or update an Opportunity. A Pipeline stage shows where that Opportunity sits in the process.
