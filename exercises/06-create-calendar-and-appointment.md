# Exercise 06: Create Calendar and Appointment

## Goal

Understand how Calendars and Appointments fit into the HighLevel lead journey.

## Estimated Time

60 minutes

## Starting Point

Start inside a test sub-account/location.

## What You Will Create

| Item Type | Exact Name to Use |
|---|---|
| Calendar | MJ Test Booking Calendar |
| Appointment | Test Appointment - MJ |

## Milestone 1: Create a Calendar

### Steps

1. Go to Calendars.
2. Find Calendar settings or create a new Calendar.
3. Create a Calendar named `MJ Test Booking Calendar`.
4. Use simple test availability.
5. Save the Calendar.

### Expected Result

You can see a Calendar named `MJ Test Booking Calendar`.

### Milestone Summary

In this milestone, you learned that:

- A Calendar controls booking availability.
- A Calendar can be used by leads or customers to book Appointments.
- Appointments can become important Workflow triggers.

### Remember This in GHL Terms

A Calendar is used for booking. An Appointment is the booked meeting connected to a Contact.

### Quick Self-Check

1. What Calendar did I create?
2. Where can I find it again?
3. What does a Calendar control?
4. How does a Calendar connect to the lead journey?
5. What could the Orchestrator inspect on Calendars?

## Milestone 2: Create or Book a Test Appointment

### Steps

1. Open the Calendar or booking view.
2. Create or book a test Appointment named `Test Appointment - MJ` if naming is available.
3. Link it to a test Contact if required.
4. Save the Appointment.
5. Review where the Appointment appears.

### Expected Result

A test Appointment exists in the Calendar or Contact activity.

### Milestone Summary

In this milestone, you learned that:

- An Appointment is a booked meeting.
- Appointments usually connect to a Contact.
- Appointment status can affect follow-up and Workflow logic.

### Remember This in GHL Terms

A Contact can book an Appointment through a Calendar. Workflows can use Appointment activity to send reminders or move Opportunities.

### Quick Self-Check

2. Which Calendar is it connected to?
3. Which Contact is it connected to?
4. How could an Appointment affect a Pipeline?
5. What Appointment issue could the Orchestrator inspect?
