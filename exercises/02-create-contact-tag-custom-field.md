# Exercise 02: Create Contact, Tag, and Custom Field

## Goal

Create basic Contact data and practice the GHL terms Contact, Tag, and Custom Field.

## Estimated Time

60 to 75 minutes

## Starting Point

Start inside a test sub-account/location.

## What You Will Create

| Item Type | Exact Name to Use |
|---|---|
| Contact | Test Lead - MJ |
| Tag | ghl-test-lead |
| Custom Field | Service Interest |

## Milestone 1: Create a Test Contact

### Steps

1. Go to Contacts.
2. Click the button to add or create a Contact.
3. Enter First Name: `Test Lead`.
4. Enter Last Name: `MJ`.
5. Use a safe test email only, such as `mj.test@example.com`.
6. Save the Contact.

### Expected Result

You can see a Contact named `Test Lead - MJ` or `Test Lead MJ` in Contacts.

### Milestone Summary

In this milestone, you learned that:

- A Contact is the central person record in HighLevel.
- A Contact can represent a lead, prospect, customer, or test person.
- Most communication, automation, and opportunity tracking connects back to the Contact.

### Remember This in GHL Terms

A Contact is the main person record in HighLevel. Do not call it a Salesforce Lead when documenting your GHL learning.

## Milestone 2: Add a Tag

### Steps

1. Open the Contact `Test Lead - MJ`.
2. Look for Tags.
3. Add the Tag `ghl-test-lead`.
4. Save if required.

### Expected Result

The Contact shows the Tag `ghl-test-lead`.

### Milestone Summary

In this milestone, you learned that:

- A Tag labels or groups a Contact.
- Tags can help segment Contacts.
- Tags are often used by Workflows for automation logic.

### Remember This in GHL Terms

A Tag is a simple but powerful label on a Contact. A Workflow can use Tags to identify or route Contacts.

### Quick Self-Check

1. What Tag did I add?
2. Where can I see the Tag?
3. How could a Workflow use this Tag?
4. Why should Tags be named clearly?
5. What Tag problems could the Orchestrator inspect?

## Milestone 3: Create a Custom Field

### Steps

1. Go to Settings.
2. Find Custom Fields.
3. Create a new Custom Field named `Service Interest`.
4. Use a simple text field unless your account asks for a field type.
5. Save the Custom Field.
6. Return to the Contact and enter a value such as `CRM Audit` in `Service Interest`.

### Expected Result

The Contact has a Custom Field named `Service Interest` with the value `CRM Audit`.

### Milestone Summary

In this milestone, you learned that:

- A Custom Field stores extra information.
- Custom Fields help qualify and segment Contacts.
- Workflows can use Custom Field values for decisions or actions.

### Remember This in GHL Terms

A Custom Field stores extra Contact information. A Contact can have Tags and Custom Fields that help Workflows decide what to do next.
